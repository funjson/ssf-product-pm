## 2. 页面级原型 Prompt

**生成说明**：每个页面一个 Prompt。Prompt 必须包含页面 ID、页面目标、核心组件、交互状态和负向约束。

# SCR-xxx 页面名称 Prompt

```text
请生成页面：SCR-xxx <页面名称>。
页面目标：<说明页面要帮助用户完成什么任务>。
端类型：<移动端 / Web / 小程序 / 桌面端>。
目标用户：<ROLE-xxx>。
关联功能：<FEAT-xxx>。

页面布局：
- <区域 1：目标、包含组件>
- <区域 2：目标、包含组件>

核心组件：
- <CMP-xxx：组件类型、展示文案、用途>

交互要求：
- <点击 / 输入 / 上传 / 跳转 / 弹窗 / 校验>

状态要求：
- 默认态：
- 加载态：
- 空态：
- 错误态：
- 禁用态 / 无权限态：

视觉要求：
- <风格、密度、层级、强调方式>

不要生成：
- <与需求无关的营销内容、无关图标、复杂背景、未定义按钮等>
```

## 3. 流程级原型 Prompt

**生成说明**：用于生成可点击流程，必须引用页面 ID 和导航关系。

```text
请生成一个可点击流程原型。
流程名称：FLOW-xxx。
关联功能：FEAT-xxx。
页面顺序：SCR-001 → SCR-002 → SCR-003。
关键交互：
1. 用户在 SCR-001 点击 CMP-xxx 后进入 SCR-002。
2. 用户在 SCR-002 完成 <动作> 后进入 SCR-003。
异常路径：
- 当 <条件> 时，在当前页面展示 <错误表现>。
```

## 6. 原型与规格一致性检查

**生成说明**：生成原型后，AI 应用此清单检查原型是否偏离规格。

| check_id | 检查项 | 通过标准 | 结果 |
|---|---|---|---|
| CHECK-001 | 页面是否全部来自 UI 页面清单 | 不得出现未定义页面 |  |
| CHECK-002 | 组件是否全部有 component_id | 关键组件必须可追踪 |  |
