# ssf-workspace 说明

> 本目录是 `ssf-product-pm` 在当前工作区中的流程实例管理目录。它用于同时管理多个彼此隔离的产品设计事项，避免新事项覆盖旧事项。

## 1. 目录职责

- 记录当前工作区内所有产品设计实例。
- 帮助 AI 判断用户请求是新流程、旧流程续跑、单文档执行，还是基于旧基线的变更。
- 保存每个实例的独立 `manifest.md`、产品规格文档和变更记录。

## 2. 放什么

- `index.md`：全局实例索引，所有实例必须登记。
- `instances/`：每个独立事项一个子目录。

## 3. 不放什么

- 不放人员排期、工时、绩效、项目管理审批。
- 不放数据库设计、接口路径、缓存、消息队列等架构实现细节。
- 不把多个无关产品混写在同一个实例中。

## 4. Agent 使用规则

1. 写任何产品规格文件前，必须先读取 `index.md`。
2. 如果用户请求与现有实例身份一致，进入该实例续跑或变更。
3. 如果用户请求与现有实例明显不一致，必须新建实例或请求用户确认。
4. 不确定是否同一实例时，默认不覆盖旧实例。
5. 用户要求重新生成、覆盖、跳阶段或变更时，必须先执行 intake gate。
