# instances 目录说明

> 本目录用于保存多个彼此隔离的产品设计实例。每个实例对应一个独立产品、独立功能事项或独立变更任务。

## 1. 目录职责

- 一个子目录代表一个独立实例。
- 每个实例必须有自己的 `manifest.md`。
- 每个实例的产品规格文档必须写在自己的 `product-spec/` 下。

## 2. 命名规范

推荐使用稳定实例 ID：

```text
SPI-001
SPI-002
SPI-003
```

也可以在目录名中追加短名称，但必须保留实例 ID：

```text
SPI-001-dailybrief
SPI-002-light-health
```

## 3. Agent 使用规则

- 不允许把两个无关事项写入同一个实例。
- 不允许因当前实例最近活跃就默认覆盖。
- 若用户请求像是新事项，必须先回到 `ssf-workspace/index.md` 做判断。
