# product-spec 目录说明

> 本目录保存当前实例的产品设计文档。它只属于当前实例，不代表整个工作区的唯一产品规格。

## 1. 文档顺序

```text
01-research-insight.md
02-requirement-analysis.md
03-prd.md
04-product-architecture.md
05-feature-task-spec.md
06-ui-ia-screen-inventory.md
07-structured-ui-interaction-spec.md
08-prototype-prompt-ui-annotation.md
09-product-baseline-change.md
```

## 2. 使用规则

- 先调研，再需求分析，再 PRD，再产品架构，再功能任务规格。
- `intake.md` 位于实例根目录，不属于 product-spec 版本文档。
- 单独生成某个文档时，必须在文档中标记缺失的上游输入。
- 修改已有规格时，必须同步更新 `09-product-baseline-change.md`。
- 每个关键对象必须使用稳定 ID，并能跨文档追踪。
